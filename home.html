Iterative Design

Overview

Startup of our choice--Gather--helps people and team coordinate with smart & automated communication by deep integration with Slack, so employees could connect with each other more easily with more streamlined message sending based on special events such as onboarding/offboarding, work anniversary, birthdays, etc.

For Gather, we are designing an application that is suitable for desktop manipulation, since our target audience would be office workers who need to be on their computer to monitor and manage their working progress. We believe that those who work in large company and have complicated projects that need collaboration between departments would benefit a lot from this app, especially due to the large amoung of functionalities the app provides as well as a strong slack integration.

Design Principles & Inspirations

In our design, we wish that the main functionalities be centered around two modules--workspace & personal dashboard.

Workspace: forming connections with company employees, automated slack message management system, team collaboration on projects.

Personal dashboard focuses on sorting out personal tasks and duties from workspace.

To release stress in the workplace, we want to use color palettes with low saturation colors, but have primary colors to highlight important messages. Increase organization and tidiness of layout, use hierarchical navbar.

Our inspiration of design (when brainstorming) comes from Slack, Notion, Trello, Cloze, and Facebook Workplace.


Mockup Crit in Studio

Gianna Giancarlo (Brown CS alum, VP Experience Strategy at Hero Digital)

  • Users normally have the impression of the layout of the app on first sight, so it was rather surprising that our homepage’s navbar looks a bit different from the navbar in the other pages. In order to keep up with visual consistency, it is suggested that we keep the homepage navbar layout roughly the same as all other pages to increase learnability.

  • On the homepage, the information in each card was pretty messy in the first place, sometimes with the same font size highlighting different information (e.g. name and event both have larger font size but it’s hard to tell the emphasis when highlights are inconsistent). To improve upon that, having the same level of information having the same font size can help with users processing information more quickly. Icons could be useful in building categories, but sometimes text hierarchy, if meticulously designed, can achieve the same effect.

Feedback from students

  • Make it clearer what are clickable and what are not (e.g. navbar)
  • Add buttons or icons for direct tutorial (increase learnability, since it is said that the app has a lot of functions and might be a bit hard for beginners to learn.
  • Make company announcements on the homepage impermanent
  • Improve color contrast (e.g. add shadow/outline) in planning page to serve a wide range of users (increase accessibility)

Analysis of Results

Homepage overview task

Task Description:

The users are asked to explore the homepage and describe their thoughts.

Users’ reactions:

Some users could identify the prototype as a platform that is designed for a company-wide dashboard, which is consistent with our design idea. Besides, they provided some compliments on the well organized layout and structure, reminding us on the right track. Since not everyone of them were familiar with Slack, it is hard for them to identify the main purpose of the slack dashboard.

Connection task

Task Description:

The major emphasis of the connection task is to both build new connections within the company, and at the same time keep track of existing connections. To achieve this goal, two subtasks are created so users can achieve the main goal of keeping connections organized.

Subtask 1: The users are asked to build new connections through the searching functionality, find the software engineer L.R., and add her to existing connections.

Subtask 2: The users are asked to find out more information about the existing information of one of his/her connections through exploring the existing connection network. Specifically, users are expected to find Jose G. from the marketing department and check out his email address.

Users’ reactions:

User testing: Among the three users we tested, most of the users were able to find L.R. and add her to existing connections, and also find Jose.G to check out his email. To find L.R, most of the users clicked on the search bar and look for software engineer L.R., then adding her to the connection by clicking on the add icon with “Add to personal connections.” All users are able to find Jose through clicking on the Marketing bubble in the personal connections page and navigate to his email address. This result validates our intention of building the page in two parts so users can both find new connections and also manage existing connections. One user also said she liked the layout of the bubble personal connections.

Errors: Some users were having some trouble finding the specific persons in the task in the first place. This error might be a result of not knowing which page is the person’s existing connections and where to find new connections. One of the users wasn’t able to find L.R. since she didn’t notice the search bar was clickable and kept on searching for L.R. in the personal connections. To improve upon this user error, we might consider adding text instructions within the search bar indicating that the search bar allows user to search for both new and existing connections; if the user is an existing connection, the page will direct the user to the profile like that of Jose, if the person searched for is new, the page will provide a holistic result page of all people that matches information in the search bar.

Calendar task

Task Description:

This section allows users to explore the personal dashboard. The task is to check out events happening on a given day, as well as its participants on the calendar from a personal dashboard.

Users’ reactions:

One of the users had trouble finding the calendar since it does not show in the navbar, and later she navigated to the planning section by chance and thought it might be the calendar she was finding since it takes the layout of a calendar for the team. The rest of the users noticed the calendar on the homepage under the section of the personal dashboard and successfully navigated to the page. They quickly found the events happening on the given day which was highlighted and clicked on the drop down menu to check the participants. All of the users completed this task, although one of them took some time finding the calendar.

Project task

Task Description:

Project task: Main task: This task aims to allow audiences to check the progress of different projects. Subtask: The users are asked to check the progress of project 1, which is one of your priority projects. Meanwhile, the users could list the tasks which are in progress.

Users’ reactions:

All of the three users from UserTesting successfully navigated to the main project page through the menu bar on the left. In order to fulfill the subtask, they all clicked on the hand icon which directed them to the page of the project 1 to see all the detailed information and find the correct list of tasks that were in the progress. The whole process moved smoothly and no one made any mistakes. The results from three participants are inline with our expectation, since the project page is straightforward and the layout of the main project page is clear for audiences to identify their target projects.

Potential Interface Changes

  • More obvious search bar on the connection page (or adding text indicating that the user can search for existing and new users)
  • Having a better indication that the bubble connection is existing personal connections
  • Make the calendar more noticeable on the homepage. For example, add a calendar icon under the personal dashboard section for direct tutorial.
  • Users seem to like to go back to home page a lot to navigate to other pages, it would also be nice to enable the function to handle simple tasks directly from the homepage
